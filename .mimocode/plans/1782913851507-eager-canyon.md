# План: Система продвижения (Продвижение постов + Баннеры)

## Контекст

Мебельный портал. Пользователи — мебельные компании, специалисты, поставщики, производители. Клиенты — заказчики мебели. Сейчас нет платёжной системы. Продвижение будет платным, но оплата — ручная (через инструкцию + номер телефона). Два типа продвижения:
1. **Продвижение постов** — поднятие в ленте + визуальная метка "Продвижение"
2. **Баннеры** — рекламные баннеры в ленте и в каталоге изображений

---

## Этап 1: Модели базы данных (Prisma)

**Файл:** `prisma/schema.prisma`

### Promotion (продвижение постов)
```prisma
model Promotion {
  id        String   @id @default(cuid())
  status    String   @default("pending") // pending | active | expired
  startDate DateTime @default(now())
  endDate   DateTime
  createdAt DateTime @default(now())

  post   Post @relation(fields: [postId], references: [id], onDelete: Cascade)
  postId String
  user   User @relation(fields: [userId], references: [id], onDelete: Cascade)
  userId String

  @@index([postId])
  @@index([userId])
  @@index([status, endDate])
}
```

### Banner (рекламные баннеры)
```prisma
model Banner {
  id        String   @id @default(cuid())
  title     String
  imageUrl  String
  linkUrl   String
  position  String   @default("both") // feed | gallery | both
  status    String   @default("pending") // pending | active | expired
  startDate DateTime @default(now())
  endDate   DateTime
  createdAt DateTime @default(now())

  user   User @relation(fields: [userId], references: [id], onDelete: Cascade)
  userId String

  @@index([status, position])
  @@index([userId])
}
```

### Изменения в существующих моделях
- **Post** — добавить `promotions Promotion[]` (reverse relation)
- **User** — добавить `promotions Promotion[]` и `banners Banner[]` (reverse relations)

## Этап 2: API маршруты

### `/api/promotion` — управление продвижением постов

**Файл:** `src/app/api/promotion/route.ts`

| Метод | Описание |
|-------|----------|
| `GET` | Список продвижений текущего пользователя (с информацией о посте) |
| `POST` | Создать продвижение postId (7/14/30 дней). Проверка: не CLIENT, пост принадлежит пользователю, нет активного продвижения для этого поста |

### `/api/promotion/[id]` — конкретное продвижение

**Файл:** `src/app/api/promotion/[id]/route.ts`

| Метод | Описание |
|-------|----------|
| `GET` | Детали продвижения |
| `DELETE` | Отменить продвижение (только своё) |

### `/api/promotion/banners` — управление баннерами

**Файл:** `src/app/api/promotion/banners/route.ts`

| Метод | Описание |
|-------|----------|
| `GET` | Список баннеров текущего пользователя |
| `POST` | Создать баннер (title, imageUrl, linkUrl, position, duration). Проверка: не CLIENT |

### `/api/promotion/banners/[id]` — конкретный баннер

**Файл:** `src/app/api/promotion/banners/[id]/route.ts`

| Метод | Описание |
|-------|----------|
| `PUT` | Обновить баннер (только своё) |
| `DELETE` | Удалить баннер (только своё) |

### `/api/promotion/active` — публичный эндпоинт для отображения

**Файл:** `src/app/api/promotion/active/route.ts`

| Метод | Описание |
|-------|----------|
| `GET` | Активные продвинутые посты (status=active, endDate > now) + активные баннеры по позиции (query: position=feed|gallery). Публичный, без авторизации |

### `/api/admin/promotion` — управление заявками (только ADMIN)

**Файл:** `src/app/api/admin/promotion/route.ts`

| Метод | Описание |
|-------|----------|
| `GET` | Все заявки (promotions + banners) с фильтрацией по статусу. Включает info о пользователе |
| `PATCH` | Изменить статус заявки (pending → active, active → expired). BODY: `{ id, type: "promotion"|"banner", status }` |

### Изменение `/api/feed/route.ts`
Добавить в начало ленты активные продвинутые посты (не более 2-3), помеченных флагом `isPromoted: true`. Приоритет: сначала обычные посты по сортировке, но продвинутые ВСЕГДА сверху.

## Этап 3: Фронтенд — Страница продвижения в личном кабинете

**Файл:** `src/app/dashboard/promotion/page.tsx` (полная переработка заглушки)

### Структура страницы (3 секции):

---

### Секция 1: «Создать заявку на продвижение»

Единая форма с галочками — пользователь выбирает что хочет:

```
┌─────────────────────────────────────────────────┐
│  Создать заявку на продвижение                  │
│                                                 │
│  ☐ Продвижение поста                            │
│    ├─ Выберите пост: [dropdown из постов юзера]  │
│    └─ Срок: [7 дней] [14 дней] [30 дней]        │
│                                                 │
│  ☐ Баннер в ленте                               │
│    ├─ Название: [_______________]                │
│    ├─ Изображение: [загрузить]                   │
│    ├─ Ссылка: [_______________]                  │
│    └─ Срок: [7 дней] [14 дней] [30 дней]        │
│                                                 │
│  ☐ Баннер в каталоге изображений                │
│    ├─ Название: [_______________]                │
│    ├─ Изображение: [загрузить]                   │
│    ├─ Ссылка: [_______________]                  │
│    └─ Срок: [7 дней] [14 дней] [30 дней]        │
│                                                 │
│  Итого: XXX руб. (заглушка)                     │
│                                                 │
│  [ Отправить заявку ]                           │
└─────────────────────────────────────────────────┘
```

Логика:
- Галочки показывают/скрывают соответствующие блоки формы
- Поля обязательны только для отмеченных галочек
- Каждый выбранный пункт добавляется в заявку
- Заявка создаётся со статусом `pending` (ожидает подтверждения/оплаты)

---

### Секция 2: «Мои заявки»

Таблица всех заявок пользователя:

| Тип | Детали | Срок | Статус | Дата |
|-----|--------|------|--------|------|
| Пост «Кухни Лофт» | 14 дней | Активно | 01.07.2026 |
| Баннер в ленте «Скидки» | 7 дней | Ожидает | 30.06.2026 |
| Баннер в каталоге | 30 дней | Истекло | 15.06.2026 |

Статусы: `pending` (ожидает оплаты) → `active` (активно) → `expired` (истекло)
Кнопка «Отменить» для pending и active заявок.

---

### Секция 3: «Оформить подписку / Связаться»

- Инструкция: «Для оплаты и активации продвижения свяжитесь с нами»
- Номер телефона (кнопка tel: ссылка)
- Описание тарифов (заглушка):
  - Продвижение поста: X руб./неделя
  - Баннер: Y руб./неделя
- Кнопка «Позвонить»

## Верификация

1. `npx prisma db push` — проверить что модели создаются без ошибок
2. `npm run build` — проверить что проект собирается
3. Ручное тестирование:
   - Зайти как USER/COMPANY → личный кабинет → Продвижение → создать заявку с галочками → проверить что заявка появилась со статусом "Ожидает"
   - Зайти как ADMIN → админ-панель → Продвижение → активировать заявку → проверить что статус изменился на "Активно"
   - Проверить что продвинутый пост появился в ленте с меткой "Продвижение"
   - Проверить что баннер отображается в ленте и каталоге
   - Зайти как CLIENT → вкладка "Продвижение" не видна
   - Проверить что API `/api/promotion/active` возвращает данные без авторизации
