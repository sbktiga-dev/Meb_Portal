# План доработок (финальный)

## 1. Кнопка "Редактировать профиль" ведёт на логин

**Причина:** `<a href="/dashboard/profile">` — это обычный тег, делает полный переход. Middleware проверяет cookie `token`. Если cookie истёкла или не передаётся — редирект на `/login`.

**Исправление:** Заменить `<a href>` на `<Link href>` (Next.js клиентский роутинг, без полной перезагрузки).

**Файл:** `src/app/dashboard/settings/page.tsx` (строка 150)

---

## 2. Мобильное меню — нет выхода и настроек

**Проблема:** В меню "Ещё" (MobileBottomNav) нет кнопки "Выйти" и "Настройки".

**Исправление:** Добавить в `src/components/MobileBottomNav.tsx`:
- Настройки → `/dashboard/settings`
- Выйти → `localStorage.removeItem('token')`, cookie delete, redirect `/`

---

## 3. Регистрация без верификации email

**Новая логика:**
- Регистрация → сразу войти (без `needVerify`)
- Верификация email — опционально, из профиля/настроек
- Подписки требуют `emailVerified: true`

**Файлы:**
- `src/app/api/auth/login/route.ts` — убрать проверку `emailVerified` (строка 40-42)
- `src/app/api/auth/register/route.ts` — убрать `needVerify`, выдать токен сразу
- `src/app/register/page.tsx` — редирект на `/dashboard` после регистрации
- `src/app/dashboard/settings/page.tsx` — добавить блок "Подтвердите email"
- `src/app/api/subscription/route.ts` — проверять `emailVerified: true`
- SQL: `UPDATE "User" SET "emailVerified" = true WHERE "emailVerified" = false;`

---

## 4. Смена email в профиле (через email-код)

**Файлы:**
- `src/app/dashboard/settings/page.tsx` — секция "Смена email"
- `src/app/api/auth/change-email/route.ts` — новый API:
  - POST: генерирует 6-значный код, отправляет на новую почту
  - PUT: проверяет код, меняет email
- `prisma/schema.prisma` — добавить `emailChangeCode String?` в User
- SQL: `ALTER TABLE "User" ADD COLUMN IF NOT EXISTS "emailChangeCode" TEXT;`

---

## 5. Мониторинг: admin/stats 500 ошибка

**Причина:** Нестабильный ответ от Neon DB (таймауты при cold start).

**Решение:** Добавить try-catch с fallback значениями в `src/app/api/admin/stats/route.ts`.
