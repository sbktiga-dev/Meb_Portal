## КРИТИЧНЫЕ ПРОБЛЕМЫ (29) — требуют немедленного исправления

### Безопасность / Авторизация

1. **JWT role bypass в stats + posts API** — `admin/stats/route.ts:16`, `admin/posts/route.ts:16` читают `payload.role` из JWT вместо запроса в БД. Смещённый админ сохраняет доступ до истечения токена (до 7 дней).
   - **Фикс**: Заменить на `prisma.user.findUnique({ where: { id }, select: { role: true } })`

2. **Admin self-promotion** — `admin/users/[id]/role/route.ts:23` — нет проверки `params.id === payload.userId`. Админ может повысить себя или любого до ADMIN.
   - **Фикс**: `if (params.id === payload.userId) return 400`

3. **Admin self-ban** — `admin/users/[id]/ban/route.ts:17` — нет проверки на само-бан. Админ может заблокировать себя.

4. **Math.random() для security-кодов** — `change-email/route.ts`, `verify-email/route.ts` используют `Math.random()` для генерации кодов. Предсказуемые коды.
   - **Фикс**: `crypto.randomInt()` или `crypto.randomBytes()`

5. **Password reset token не инвалидируется** — `forgot-password/route.ts` — старые токены остаются активными. Можно запросить несколько.

### Утраченные данные / Сломанные эндпоинты

6. **Documents/[id] API route отсутствует** — `documents/[id]/page.tsx:45` вызывает `/api/documents/${id}`, но эндпоинта нет. Просмотр и скачивание документов сломаны на 100%.

7. **Conversations/[id] route отсутствует** — `messages/[id]/page.tsx:44` вызывает `/api/conversations/${id}`, но эндпоинта нет. Страница чата не загружает метаданные диалога.

8. **Supplier + Manufacturer DELETE не существуют** — Админ-кнопки "Удалить" вызывают DELETE, который возвращает 405. UI удаляет строку, но данные остаются в БД.

### Утечка данных

9. **Private group members leaking** — `api/groups/[id]/route.ts:9-20` — приватные группы отдают полный список участников анонимным пользователям.

10. **Active promotions leaks user data** — `api/promotion/active/route.ts:6,28,36` — публичный эндпоинт отдаёт `content` и `author.role` анонимам.

11. **Phone exposure** — API `/api/users/[id]` и `/api/users/[id]/profile` возвращают `phone` всем. UI скрывает для гостей, но данные доступны через прямой fetch.

### Функциональные баги

12. **Analytics period selector — косметический** — `analytics/page.tsx:21-23` — `period` state переключается, но `useEffect` с пустым deps `[]` — данные не перезагружаются.

13. **Analytics показывает все портфолио** — `analytics/page.tsx:34,49` — `/api/portfolio` возвращает все опубликованные элементы, не только текущего пользователя.

14. **Schema mismatch: premium plan** — `subscription/route.ts:53` принимает `'lite', 'pro', 'premium'`, но schema объявляет только `"lite" | "pro"`.

15. **Promotion POST — denylist вместо allowlist** — `promotion/route.ts:47-48` — проверяет `role === 'CLIENT'`, пропуская неизвестные роли.

16. **Forgot-password не отправляет email** — `forgot-password/route.ts` — нет вызова `sendEmail()`. Функциональный баг: сброс пароля не работает.

17. **getUserFromToken не проверяет banned** — `lib/auth.ts` — забаненные пользователи сохраняют API-доступ до истечения JWT.

18. **Registration multi-step без транзакции** — `register/route.ts` — создание User + Company/Supplier без `prisma.$transaction()` — сиротские записи при ошибке.

19. **PrismaClient singleton в production** — `lib/prisma.ts` — кэширует только в dev. В production создаёт новый клиент → leak соединений.

20. **Hardcoded JWT fallback** — `middleware.ts:5` — `|| 'fallback-secret'` при отсутствии env vars = обход авторизации.

21. **16/40 компонентов без dark mode** — SpecialistCard, SupplierCard, CompanyCard, ImageCard, DocumentCard, FilterPanel, Pagination, SearchBar, RefTable, Modal, Loading, ImageGrid, ContactInfo, BannerAd, PromotionBadge — яркие карточки на тёмном фоне.

22. **Нет error.tsx** — ни один route segment не имеет error boundary. Runtime error = краш всего приложения.

23. **Нет not-found.tsx** — нет кастомной 404-страницы.

24. **PageSEO — бесполезен для SEO** — `PageSEO.tsx` использует `useEffect` для `document.title` — поисковики не видят мета-теги. Нужен `generateMetadata()`.

25. **10 из 13 admin страниц без клиентской проверки роли** — Только middleware как барьер. Прямой URL работает для demoted admin.

26. **Event cleanup блокирует каждый GET** — `api/events/route.ts:17-24` — `deleteMany` на каждом запросе листинга. Performance landmine.

27. **Conversations unread count — no-op** — `api/conversations/route.ts:54-56` — вычитание всегда = 0. `lastReadAt` бесполезен.

28. **Email HTML template — XSS** — `lib/email.ts:52` — `userName` вставляется без HTML-escaping.

29. **Banner PUT без валидации позиции** — `promotion/banners/[id]/route.ts:41` — create валидирует, update нет.

## MEDIUM ПРОБЛЕМЫ (88) — исправить в ближайшие спринты

### Ключевые patterns:
- **ActivityLog**: `logActivity()` вызывается только в ~5 route. Все мутации (subscription, promotion, banner, role, ban, post-moderation, feedback) не логируются.
- **Auth boilerplate**: Каждый route дублирует 8+ строк токен-извлечения. Нет shared `requireAuth()`.
- **Rate limiting**: In-memory Map — бесполезен в serverless. Redis нужен для реальной защиты.
- **Pagination**: Многие list API не ограничивают `limit` — `limit=999999` dump всего.
- **Input validation**: Многие API не валидируют длину строк, типы enum, JSON structure.
- **Download counters**: Существуют в schema, но обновляются только в React state (client-side) — все статистики ephemeral.
- **Нет уведомлений**: Group join, event join, post creation не создают notification records.
