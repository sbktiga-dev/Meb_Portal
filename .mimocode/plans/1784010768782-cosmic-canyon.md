# План: Система модерации отзывов

## Суть
Отзывы больше не публикуются сразу. Новый отзыв в статусе `pending` ожидает одобрения адресата в течение 24 часов. Адресат может одобрить или оспорить. Спорные отзывы отправляются админу на финальное решение.

## Статусы отзывов
- `pending` — ожидает одобрения адресата (до 24ч)
- `approved` — одобрен адресатом, опубликован
- `disputed` — оспорен адресатом, на рассмотрении у админа
- `rejected` — отклонён админом
- `auto_approved` — автоматически одобрен по истечении 24ч

## Изменения

### 1. Prisma-схема — `prisma/schema.prisma`

Добавить поля в `UserProfileReview`:
```
status        String    @default("pending")  // pending|approved|disputed|rejected|auto_approved
disputeText   String?                        // текст оспаривания
disputeImages String?   @default("[]")       // JSON массив URL картинок доказательств
disputedAt    DateTime?                      // когда оспорен
respondedAt   DateTime?                      // когда одобрен/осспорен
expiresAt     DateTime?                      // когда истекает 24ч на одобрение
```

**УБРАТЬ уникальное ограничение** `@@unique([reviewerId, targetUserId])` — теперь один пользователь может оставить сколько угодно отзывов другому.

### 2. Миграция — `prisma/migrations/...`

SQL:
```sql

ALTER TABLE "UserProfileReview" ADD COLUMN "disputedAt" TIMESTAMP(3);

ALTER TABLE "UserProfileReview" ADD COLUMN "expiresAt" TIMESTAMP(3);
DROP INDEX IF EXISTS "UserProfileReview_reviewerId_targetUserId_key";
```

### 3. API отзывы — создание и модерация

#### Новый файл: `src/app/api/users/[id]/reviews/route.ts`
- **POST** — создание отзыва (статус `pending`, `expiresAt = now + 24h`)
- **GET** — список опубликованных отзывов (`status IN ('approved', 'auto_approved')`)

#### Новый файл: `src/app/api/users/[id]/reviews/[reviewId]/route.ts`
- **GET** — детали отзыва (для адресата — с полной информацией)

#### Новый файл: `src/app/api/users/[id]/reviews/[reviewId]/approve/route.ts`
- **POST** — одобрение отзыва адресатом (меняет статус на `approved`)

#### Новый файл: `src/app/api/users/[id]/reviews/[reviewId]/dispute/route.ts`
- **POST** — оспаривание отзыва (адресат отправляет текст + картинки, статус → `disputed`)

## Верификация

1. `npx prisma db push` — проверить что схема применяется
2. Создать отзыв через профиль → статус `pending`
3. Зайти под адресатом → видеть отзыв на одобрение
4. Одобрить → отзыв публикуется
5. Оспорить → статус `disputed`
6. В админке → видеть спорный отзыв, одобрить/отклонить
7. Через 24ч без действия → `auto_approved`
8. Написать несколько отзывов одному пользователю → все создаются (не заменяются)
