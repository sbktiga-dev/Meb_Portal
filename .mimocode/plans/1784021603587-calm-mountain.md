### 2. SEO метаданные (27 страниц используют клиентский PageSEO)

**Проблема:** Компонент `PageSEO` использует `useEffect` для установки title/description. Поисковики не исполняют JS — метаданные не индексируются.

**Файлы с PageSEO:**
- Листинг: specialists, events, groups, feed, refs, manufacturers, suppliers, companies, documents, products, gallery
- Детальные: specialists/[id], events/[id], groups/[id], feed/[id], products/[id], documents/[id], companies/[id], manufacturers/[id], suppliers/[id], gallery/[id], profile/[userId]
- Админ: users, feedback, activity, monitoring
- Клиентские страницы: feed/new

**Подход:** Для серверных компонентов — добавить `export const metadata: Metadata`. Для клиентских ('use client') — оставить PageSEO как есть (они не могут использовать generateMetadata).

Страницы которые МОЖНО конвертировать в серверные:
- specialists, events, groups, manufacturers, suppliers, companies — загружают данные на сервере
- Детальные страницы [id] — тоже серверные (await params)

Страницы которые остаются клиентскими:
- feed (страница), products, gallery, documents, refs — используют 'use client' + InfiniteScroll
- Админ страницы — используют 'use client'

### Этап 4: Безопасность

**Что уже есть (хорошо):**
- ✅ Rate limiting (in-memory) на auth, upload, search, comments, likes
- ✅ Input sanitization (sanitizeInput) — убирает HTML теги, javascript:, data:, vbscript:, event handlers
- ✅ Authorization — role-based checks на всех admin endpoints
- ✅ Authentication — JWT + bcrypt (12 rounds)
- ✅ SQL Injection — Prisma ORM параметризует запросы
- ✅ Banned users — проверка при входе

**Что нужно исправить (проблемы):**

1. **Безопасные HTTP заголовки** — добавить в `next.config.js`:
   - `Content-Security-Policy` — защита от XSS
   - `X-Frame-Options: DENY` — защита от clickjacking
   - `X-Content-Type-Options: nosniff` — защита от MIME sniffing
   - `Referrer-Policy: strict-origin-when-cross-origin`
   - `X-XSS-Protection: 1; mode=block`

2. **Middleware для авторизации** — создать `src/middleware.ts`:
   - Проверка JWT на защищённых маршрутах (/dashboard/*, /admin/*)
   - Редирект неавторизованных пользователей
   - Защита admin routes от обычных пользователей

3. **CORS настройки** — добавить явные CORS headers для API routes

4. **Token в httpOnly cookie** (рекомендация, не критично):
   - Сейчас: localStorage → уязвим XSS
   - Лучше: httpOnly cookie → не доступен для JS
