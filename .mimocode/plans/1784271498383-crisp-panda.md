## Задача 1: Визуальный редактор постов (drag-and-drop)

### Суть
Конструктор постов с перетаскиванием блоков. Пользователь собирает пост из готовых блоков: заголовок, текст, изображение, галерея, разделитель, цитата. Каждый блок редактируется inline. Есть шаблоны-заготовки для быстрого старта.

### Новые файлы

**`src/components/PostEditor.tsx`** — основной drag-and-drop редактор

Архитектура редактора:
- Левая панель: библиотека блоков (перетаскиваются на canvas)
- Центр: canvas с блоками (порядок меняется drag-and-drop)
- Правая панель: настройки выбранного блока + превью
- Верхняя панель: выбор шаблона-заготовки

Типы блоков:
| Блок | Иконка | Описание |
|---|---|---|
| Heading | H | Заголовок (H1-H3), выравнивание |
| Text | T | Текстовый блок, rich-text (жирный, курсив, ссылки) |
| Image | Фото | Одно изображение с подписью |
| Gallery | Галерея | Сетка изображений (2-4 колонки) |
| Quote | Цитата | Цитата с указанием автора |
| Divider | --- | Горизонтальная линия-разделитель |
| Button | Кнопка | Кнопка-ссылка с текстом |

Интерфейс блока:
```typescript
interface EditorBlock {
  id: string;
  type: 'heading' | 'text' | 'image' | 'gallery' | 'quote' | 'divider' | 'button';
  content: Record<string, any>; // зависит от типа
  order: number;
}
```

Реализация drag-and-drop:
- HTML5 native drag-and-drop API (без сторонних библиотек)
- dragstart/dragover/drop на canvas
- reorder блоков внутри canvas через drag handle
- Удаление блока по крестику
- Дублирование блока по иконке копии

Интеграция с шаблонами:
- При выборе шаблона — canvas заполняется предустановленными блоками
- Пользователь редактирует текст/фото в блоках
- Можно добавлять/удалять/переставлять блоки

**`src/lib/postTemplates.ts`** — определения шаблонов

Шаблоны (наборы блоков для быстрого старта):

Для мебельщиков:
| Шаблон | Блоки |
|---|---|
| Новый товар | Heading + Image + Text(описание) + Text(цена) + Button(заказать) |
| Проект | Heading + Gallery(4 фото) + Text(описание) + Quote(отзыв клиента) |
| До/После | Heading + Gallery(2 фото: до/после) + Text(описание работ) |
| Процесс производства | Heading + Gallery(3-4 фото) + Text(этапы) |

Для поставщиков:
| Шаблон | Блоки |
|---|---|
| Новый товар | Heading + Image + Text(характеристики) + Text(цена) + Button(купить) |
| Акция | Heading + Image(баннер) + Text(условия) + Text(сроки) + Button(ссылка) |
| Доставка | Heading + Text(условия) + Text(города) + Button(контакты) |

Для специалистов:
| Шаблон | Блоки |
|---|---|
| Портфолио | Heading + Gallery + Text(описание) + Quote(клиент) |
| Обзор | Heading + Image + Text(плюсы/минусы) + Heading(итог) |
| Совет | Heading + Text + Image(опционально) |

**`src/components/editor/BlockPalette.tsx`** — панель блоков (left sidebar)
**`src/components/editor/EditorCanvas.tsx`** — canvas с блоками (center)
**`src/components/editor/BlockRenderer.tsx`** — рендер блока по типу
**`src/components/editor/BlockSettings.tsx`** — настройки выбранного блока (right sidebar)

### Дизайн

## Задача 2: Мониторинг переписок в админке

### Суть
Полный доступ администратора ко всем перепискам на платформе.

### Новые файлы

**`src/app/admin/messages/page.tsx`** — страница мониторинга

Интерфейс:
- Слева (30%): список всех диалогов
  - Аватар + Имя участников
  - Последнее сообщение (обрезка до 50 символов)
  - Время
  - Поиск по имени/email
- Справа (70%): переписка
  - Сообщения как в обычном чате
  - Read-only (без возможности отправки)
  - Метка внизу: "Просмотрено администратором"
  - Пагинация (по 50 сообщений)

**`src/app/api/admin/messages/route.ts`** — GET: все диалоги
- Возвращает: id диалога, участники, последнее сообщение, время, кол-во сообщений
- Фильтр: ?search=имя
- Пагинация: ?page=1&limit=20

**`src/app/api/admin/messages/[conversationId]/route.ts`** — GET: сообщения диалога
- Возвращает: все сообщения с авторами и timestamp
- Пагинация: ?limit=50&offset=0

**Изменить: `src/app/admin/page.tsx`**
- Добавить в `sections` массив: `{ href: '/admin/messages', label: 'Переписки', icon: ... }`

## Задача 3: Политика конфиденциальности

### Новые файлы

**`src/app/privacy/page.tsx`** — страница политики

Содержимое (без иероглифов, чистый русский):
1. Введение — что регулирует данная политика
2. Какие данные собираем — регистрация (имя, email, пароль), профиль (аватар, описание, контакты)
3. Переписки — хранятся на сервере, доступны участникам диалога и администратору платформы
4. Загрузки — файлы хранятся в Object Storage (Yandex Cloud)
5. Cookies — используются для авторизации (JWT в cookie)
6. Права пользователя — доступ к данным, удаление аккаунта
7. Безопасность — шифрование HTTPS, хеширование паролей
8. Контакты — email для вопросов
9. Дата вступления в силу
