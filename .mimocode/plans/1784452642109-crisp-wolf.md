## Задача 2: Админ badge'и

**Когда:** пользователь создаёт пост/отзыв/сообщение/etc → на карточке `/admin` появляется красная точка с числом

**Модель `AdminAlert` (Prisma):**
```prisma
model AdminAlert {
  id        String   @id @default(cuid())
  type      String   // new_user|new_post|new_review|new_message|new_feedback|new_product|new_image
  title     String
  read      Boolean  @default(false)
  createdAt DateTime @default(now())
  @@index([type, read])
}
```

**API:**
- `GET /api/admin/alerts/count` → `{ new_user: 3, new_post: 1, ... }`
- `PUT /api/admin/alerts/read` body `{ type }` — пометить прочитанными

**Создание alerts** — добавить `prisma.adminAlert.create()` в:
- `POST /api/auth/register` → new_user
- `POST /api/posts` → new_post
- `POST /api/users/[id]/reviews` → new_review
- `POST /api/conversations/[id]/messages` → new_message
- `POST /api/feedback` → new_feedback
- `POST /api/products/manage` (POST) → new_product
- `POST /api/images/upload` → new_image

**UI:** badge на карточках `/admin` + polling каждые 3 мин. При клике на карточку — read.

## Задача 3: Badge'и для ВСЕХ пользователей на навигации

**Концепция:** Когда появляется новый контент (посты, картинки, документы, события, справочники), на вкладке навигации появляется красная точка. При просмотре контента badge пропадает. На individual items тоже показывается "NEW".

### Архитектура

**1. Модель `UserSectionView` (Prisma):**
```prisma
model UserSectionView {
  id         String   @id @default(cuid())
  userId     String
  section    String   // feed|gallery|products|events|documents|refs
  lastViewed DateTime @default(now())
  @@unique([userId, section])
  @@index([userId])
}
```
Хранит最后一次 пользователя посетил каждую секцию.

**2. API `GET /api/content/badges`** (для авторизованных):
Возвращает количество НОВЫХ элементов в каждой секции после `lastViewed`:
```json
{ "feed": 5, "gallery": 3, "products": 0, "events": 1, "documents": 2, "refs": 0 }
```
Логика: `prisma.post.count({ where: { createdAt: { gt: lastViewed } } })` и т.д.

**3. API `PUT /api/content/badges/seen`** body `{ section }`:
Обновляет `lastViewed` на `now()` для данной секции. Вызывается при:
- Переходе на страницу секции (/feed, /gallery, etc.)
- Просмотре individual item (/gallery/[id])

**4. Header badge'и:**
- Header фетчит `/api/content/badges` при загрузке
- На каждой вкладке навигации показывается красная точка с числом если > 0
- Polling каждые 3 минуты
- При клике на вкладку → PUT `/api/content/badges/seen` → badge пропадает

**5. Individual items "NEW" badge:**
На страницах列表 (gallery, documents, events, etc.):
- При загрузке страницы: каждый элемент сравнивается с `lastViewed`
- Если `createdAt > lastViewed` → показывается маленький badge "NEW" на карточке
- При просмотре страницы ( mount ) → PUT seen для этой секции

**6. Куда НЕ добавляем badge'и:**
- Участники (specialists, suppliers, etc.) — не нужны, контент статичный
- Товары — опционально (по желанию)

### Вкладки с badge'ами:
| Вкладка | Что считается новым |
|---------|-------------------|
| Лента | Новые посты |
| Каталог | Новые изображения |
| Товары | Новые товары |
| Группы | (пропустить — нет смысла) |
| События | Новые события |
| Документы | Новые документы |
| Справочники | Новые справочники |
