# План: Уведомления поставщикам о новых компаниях/производствах

## Что нужно

Когда регистрируется новая компания (COMPANY) или производство (MANUFACTURER), все поставщики (SUPPLIER) с активной подпиской и включённой настройкой `notifyNewCompanies` получают in-app уведомление. Настройка вкл/выкл — на странице "Продвижение" в личном кабинете.

## Архитектура

### 1. Supplier модель — добавить поле настройки

```prisma
model Supplier {
  ... (существующие поля)
  notifyNewCompanies Boolean @default(false)  // ← НОВОЕ
}
```

### 2. Registration API — уведомления поставщикам

В `POST /api/auth/register`, после создания компании/производства:
```ts
// После prisma.company.create или prisma.manufacturer.create:
if (userRole === 'COMPANY' || userRole === 'MANUFACTURER') {
  const suppliers = await prisma.user.findMany({
    where: {
      role: 'SUPPLIER',
      supplier: { notifyNewCompanies: true },
    },
    include: { subscription: { where: { status: 'active' } } },
  });
  
  const activeSuppliers = suppliers.filter(s => s.subscription.length > 0);
  const entityType = userRole === 'COMPANY' ? 'компания' : 'производство';
  
  await prisma.notification.createMany({
    data: activeSuppliers.map(s => ({
      type: 'new_entity',
      message: `На портале зарегистрировалась новая ${entityType}: ${name || email.split('@')[0]}`,
      userId: s.id,
      link: userRole === 'COMPANY' ? '/companies' : '/manufacturers',
    })),
  });
}
```

### 3. API для переключения настройки

### 4. UI на странице "Продвижение"

Добавить секцию "Уведомления" для поставщиков:
- Показывается только если `user.role === 'SUPPLIER'`
- Toggle: "Уведомлять о новых компаниях и производствах"
- Рядом текст: "Вы будете получать уведомления, когда на портале зарегистрируется новая компания или производство"
- Если нет подписки — toggle серый (disabled) с подсказкой "Требуется подписка"
