# ПЛАН ДАЛЬНЕЙШИХ УЛУЧШЕНИЙ — МЕБЕЛЬНЫЙ ПОРТАЛ

## Текущее состояние (на 2026-06-21)

- **36 страниц**, **44 API-эндпоинта**, **23 компонента**, **18 моделей Prisma**
- Стек: Next.js 14 + React 18 + Prisma + SQLite + Tailwind CSS + JWT
- Основной функционал: каталог, лента, портфолио, подписки, уведомления, поиск, избранное, сообщения, админ-панель

---

## Фаза 4 — Улучшения UX и производительности

### 4.1 Infinite Scroll
- Замена пагинации на бесконечную прокрутку в ленте (`/feed`), галерее (`/gallery`), документах (`/documents`)
- `IntersectionObserver` для автозагрузки следующей страницы
- Скелетон-загрузка при подгрузке
- Компонент `InfiniteScroll.tsx`

### 4.2 Skeleton Loading
- Замена `animate-pulse` заглушек на реалистичные skeleton-компоненты для каждой страницы
- Skeleton для карточек постов, изображений, портфолио, профилей
- Shimmer-эффект при загрузке

### 4.3 Оптимизация изображений
- Next.js `<Image>` вместо `<img>` для автоматического WebP/AVIF
- Lazy loading + placeholder blur
- Responsive sizes для мобильных
- `next.config.js` — remotePatterns для внешних URL

### 4.4 PWA (Progressive Web App)
- `manifest.json` для установки на рабочий стол
- Service Worker для кэширования статики
- Offline-режим для просмотра ранее загруженных страниц
- Иконки для разных размеров экрана

### 4.5 Горячие клавиши
- `Ctrl+K` — глобальный поиск (уже есть)
- `Ctrl+N` — новый пост
- `Ctrl+/` — справка по горячим клавишам
- `Escape` — закрытие модальных окон
- Страница справки `/shortcuts`

## Фаза 8 — Безопасность и стабильность

### 8.1 Rate Limiting
- Ограничение API-запросов: 100 req/min для обычных, 10 req/min для auth
- Защита от brute-force на login
- Middleware + Redis или in-memory счётчик

### 8.2 CSRF-защита
- CSRF-токены для форм (login, register, create post, upload)
- SameSite cookie + проверка origin

### 8.3 Валидация на сервере (Zod)
- Zod-схемы для всех API-запросов (уже частично есть)
- Единый файл `src/lib/validations.ts`
- Валидация перед Prisma-запросами

### 8.4 Хранение паролей
- Проверка: bcryptjs уже используется ✓
- Миграция существующих паролей если нужно

### 8.5 Audit Log
- Модель `AuditLog` (action, entity, entityId, userId, timestamp)
- Логирование действий админа
- Логирование входов/выходов
