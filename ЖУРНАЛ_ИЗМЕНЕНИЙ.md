## Фаза 3 (2026-06-21) — Фото в портфолио + редактирование

### API: загрузка файлов
- **`/api/upload`** — новый эндпоинт загрузки для авторизованных пользователей (не только ADMIN)
- Сохраняет файлы в `public/uploads/`
- Валидация: макс. 10MB, только изображения (JPG, PNG, GIF, WebP)
- Генерирует уникальное имя файла

### API: редактирование
- **`/api/portfolio/[id]`** — добавлен `PUT` (обновление портфолио)
- **`/api/posts/[id]`** — добавлен `PUT` (обновление постов)
- Проверка авторства: только автор или ADMIN может редактировать

### Портфолио: загрузка фото
- **`/dashboard/portfolio/new`** — добавлена загрузка изображений
  - Превью загруженных фото с кнопкой удаления
  - Drag & drop / выбор файлов
  - Индикатор загрузки
- **`/dashboard/portfolio`** — список работ показывает реальные фото
  - Кнопка "Редактировать" при наведении на фото
  - Ссылка "Изменить" внизу карточки

### Портфолио: редактирование
- **`/dashboard/portfolio/[id]/edit`** — страница редактирования работы
  - Загрузка текущих данных из API
  - Загрузка новых фото + удаление существующих
  - Сохранение изменений через PUT

### Посты: редактирование
- **`/feed/[id]/edit`** — страница редактирования поста
  - Категория, заголовок, содержание, фото, теги
  - Загрузка новых фото через `/api/upload`
- **`/feed/[id]`** — кнопка "Редактировать" для автора поста
  - Показывается только если текущий пользователь = автор

### feed/new
- Обновлён эндпоинт загрузки: `/api/images/upload` → `/api/upload`

## Фаза 3 (2026-06-21) — Лента новостей + Вертикальный формат

### Главная страница (`/`)
- Секция "Лента новостей" — последние посты
- Вертикальный формат (max-w-2xl, по центру)
- Одно главное фото с счётчиком (1/3)
- Широкие кнопки лайка/комментариев
- Градиентные аватары
- Категории с цветными бейджами

### Страница ленты (`/feed`)
- Вертикальный формат как в Instagram/Facebook
- Каждый пост на всю ширину (max-w-2xl)
- Фильтрация по категориям (Все/Новости/Проекты/Статьи/Товары)
- Сортировка (Новые/Популярные/Обсуждаемые)
- Пагинация

### Раскладка фото в постах
- 1 фото → на всю ширину (aspect-ratio 4:3)
- 2 фото → две колонки квадратом
- 3 фото → одна большая слева + две маленьких справа
- 4+ фото → сетка 2x2 с счётчиком "+N"

### Детали поста (`/feed/[id]`)
- Галерея изображений (сетка или одно фото)
- Лайтбокс при клике на фото
- Навигация по точкам в лайтбоксе
- Кнопка лайка с анимацией
- Градиентные аватары
- Комментарии с аватарами
- Кнопка "Поделиться"

### Создание поста (`/feed/new`)
- Загрузка изображений (макс. 10)
- Превью загруженных фото
- Категории с цветными кнопками
- Теги через запятую

### Тестовые данные
- 7 изображений в `public/test-images/` (JFIF формат)
- 5 тестовых постов с реальными фото
- 7 изображений в таблице Image
- Скрипт seed: `scripts/seed-test-posts.js`

### Реальные фото в карточках
- `<img>` теги вместо SVG-плейсхолдеров
- `object-cover` для правильного отображения
- `loading="lazy"` для ленивой загрузки
- Аспект-ratio через `padding-bottom` для пропорций

## Технические детали

### Стек
- Next.js 14 + React 18
- Prisma + SQLite
- Tailwind CSS
- JWT (jsonwebtoken + bcryptjs)

### База данных
- Файл: `prisma/dev.db`
- Модели: User, Post, Comment, PostLike, PortfolioItem, Image, Document, Supplier, Company, Specialist, Manufacturer, Product, Rating, Download

### Директории
- `src/app/` — страницы (routes)
- `src/app/api/` — API роуты
- `src/components/` — переиспользуемые компоненты
- `src/lib/` — утилиты (prisma, auth)
- `public/` — статические файлы
- `public/test-images/` — тестовые изображения
- `prisma/` — схема и миграции

### Правила
- SQLite (не PostgreSQL)
- `String[]` → `String` с JSON
- Enum → String
- `npx prisma db push` (не migrate)
- PowerShell без `&&`
